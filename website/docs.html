Authentication

Every API request needs your personal API key to verify who you are and which plan you're on.

Getting Your API Key

  1. Sign up for free — no credit card needed
  2. Open your Dashboard
  3. Click the API Keys tab
  4. Click Generate Key — your key starts with nrs_
  5. Copy and store it somewhere safe — it won't be shown again
Keep your key private. Don't paste it into public code, GitHub, or share it with others. If it's ever exposed, delete it from the dashboard and generate a new one.

Using Your API Key

Add it to the Authorization header of every request, like this:

Authorization: Bearer nrs_your_api_key_here

Verify Content

Detect hallucinations in AI-generated or human content before you publish. Returns flagged claims, a verifiable verdict, E-E-A-T scoring, and a Certificate of Accuracy you can share as proof.

The trust mechanism: When a claim contradicts our source data, we return correct_value — the actual correct fact. Your user checks it and finds we were right. That's the moment they trust the product.
POST /api/v1/verify

Verify AI-generated or human content for factual accuracy and E-E-A-T signals.

Request

{ "content": "OpenAI was founded in 2015. The treatment requires 500mg daily for 30 days.", "url": "https://yoursite.com/article" // optional — for the certificate record }

Response

{ "success": true, "data": { "certificate_id": "cert_a1b2c3d4", "certificate_url": "https://naraseoai.onrender.com/api/v1/verify/cert_a1b2c3d4", "summary": { "total_claims": 8, "flagged": 2, "safe": 6, "risk_score": 25, "verdict": "review_needed" // clean | review_needed | high_risk }, "eeat": { "score": 40, "grade": "moderate", "missing": ["hasAuthorMention", "hasCitations"] }, "flagged_claims": [ { "claim": "treatment requires 500mg daily for 30 days", "status": "contradicted", "risk": "high", "correct_value": "Dosage varies by patient weight and condition — no standard universal dose", "source": "Naraseo verification database", "reason": "Specific dosage claim with no citation — high liability risk" } ] } }

MCP Server Integration

Give Claude, Cursor, and Windsurf native SEO tools

What is MCP?

MCP (Model Context Protocol) lets AI tools like Claude Desktop and Cursor call Naraseo AI's SEO tools natively — as if they were built-in features. Instead of you copy-pasting data, your AI agent can audit, research, and fix SEO on its own.

Setup for Claude Desktop

  1. Open your Claude Desktop config file:
    Windows: %APPDATA%\Claude\claude_desktop_config.json
    Mac: ~/Library/Application Support/Claude/claude_desktop_config.json
  2. Add this block inside the file (create the file if it doesn't exist):
{ "mcpServers": { "naraseo-ai": { "command": "npx", "args": ["mcp-remote", "https://naraseoai.onrender.com/mcp/sse"] } } }
  1. Save the file and restart Claude Desktop
  2. In Claude, just ask: "What are the SEO issues on example.com?" — it will use Naraseo AI automatically
Works with: Claude Desktop, Cursor, Windsurf, and any MCP-compatible AI tool. No API key needed for MCP — the connection is handled automatically.

Available Tools (what your AI agent can do)

  • seo_audit — Full SEO audit with score, grade, and issues
  • solve — One call: audit + keywords + AI-generated fixes
  • solve_site — Full site audit and fix plan via sitemap
  • keyword_research — Keyword opportunities and content gaps
  • schema_validate — Validate and generate structured data
  • site_crawl — Crawl every page, find broken links and issues
  • geo_grid — Local rank tracking across a geographic grid
  • local_seo_audit — Local SEO signals and consistency check
  • competitor_analysis — Gap analysis with domain authority data
  • seo_chat — Conversational SEO assistant with page context

Error Handling

Handle API errors gracefully

HTTP Status Codes

Code Meaning Solution
200 Success Parse the response
400 Bad Request Check URL format and required fields
401 Unauthorized Verify API key is correct and included in header
403 Forbidden Feature not available on your plan
429 Rate Limited Wait before next request or upgrade plan
502 Bad Gateway URL unreachable or returns invalid HTML
500 Server Error Try again in a moment, contact support

Error Response Format

{ "success": false, "error": { "code": "FEATURE_UNAVAILABLE", "message": "Feature 'crawl' not available on Free tier", "requiredTier": "Pro" } }

Python Error Handling

import requests try: response = requests.post( 'https://naraseoai.onrender.com/api/v1/audit', headers={'Authorization': f'Bearer {api_key}'}, json={'url': url}, timeout=30 ) response.raise_for_status() data = response.json() if not data.get('success'): error = data.get('error', {}) print(f"API Error: {error.get('code')} - {error.get('message')}") return print(f"Score: {data['data']['score']}") except requests.exceptions.Timeout: print("Request timeout - try again") except requests.exceptions.ConnectionError: print("Cannot connect to API server") except requests.exceptions.HTTPError as e: if e.response.status_code == 429: print("Rate limited - upgrade plan") elif e.response.status_code == 401: print("Invalid API key") else: print(f"HTTP Error {e.response.status_code}")

Rate Limits & Quotas

Fair usage limits to ensure platform stability

Per-Tier Limits

Plan Per Minute Per Day
Free 10 requests 100 requests
Pro 60 requests 1,000 requests
Agency 200 requests Unlimited

Response Headers

Every response includes rate limit info:

X-RateLimit-Limit: 10 X-RateLimit-Remaining: 7 X-RateLimit-Reset: 1234567890

What Counts Toward Limits

  • Each API call = 1 request (regardless of what endpoint)
  • Site crawl with 50 pages = 50 requests
  • Geo-grid with 9 points = 9 requests

Handling Rate Limits (Python)

import requests import time def audit_with_retry(url, api_key, max_retries=3): for attempt in range(max_retries): response = requests.post( 'https://naraseoai.onrender.com/api/v1/audit', headers={'Authorization': f'Bearer {api_key}'}, json={'url': url} ) if response.status_code == 429: # Rate limited - wait and retry wait_time = int(response.headers.get('X-RateLimit-Reset', time.time() + 60)) - int(time.time()) print(f"Rate limited. Waiting {wait_time}s...") time.sleep(max(1, wait_time)) continue return response.json() raise Exception("Max retries exceeded") result = audit_with_retry('https://example.com', 'YOUR_API_KEY')
Need higher limits? Upgrade your plan or contact support for custom limits.
